# دليل الأمان - نظام الرسائل

## نظرة عامة على الأمان

هذا النظام مصمم بأمان عالي جداً لحماية الرسائل المهمة. إليك التفاصيل الكاملة:

## 🔒 طبقات الحماية

### 1. التشفير
- **تشفير XOR محسّن**: جميع الرسائل مشفرة قبل التخزين
- **مفتاح تشفير قابل للتخصيص**: استخدم `MESSAGE_ENCRYPTION_KEY` في متغيرات البيئة
- **تشفير Base64**: طبقة إضافية من الحماية

### 2. التحقق من البيانات
- **التحقق من صحة البريد الإلكتروني**: نمط صارم للتحقق من صيغة البريد
- **التحقق من الطول**: حدود صارمة لكل حقل
- **التحقق من المحتوى**: منع الأحرف الخطرة

### 3. الحماية من البريد العشوائي
- **كشف الكلمات الممنوعة**: قائمة بـ 30+ كلمة ممنوعة
- **فحص المحتوى**: تحليل شامل للرسالة
- **منع الروابط المشبوهة**: كشف الروابط الضارة

### 4. الحماية من الهجمات
- **Rate Limiting**: 5 رسائل كحد أقصى في الدقيقة
- **Daily Limit**: 50 رسالة كحد أقصى يومياً
- **IP Blocking**: حظر تلقائي للعناوين المخالفة
- **Honeypot**: حقل مخفي لفحص البوتات

### 5. التطهير (Sanitization)
- **إزالة HTML**: منع هجمات XSS
- **إزالة JavaScript**: منع حقن الكود
- **إزالة البيانات الخطرة**: حماية من SQL Injection

## 🛡️ إعدادات الأمان

### متغيرات البيئة المطلوبة

```env
# تشفير الرسائل
MESSAGE_ENCRYPTION_KEY=your-secure-key-here-change-in-production

# تخزين Upstash Redis (موصى به)
UPSTASH_REDIS_REST_URL=https://your-redis-url.upstash.io
UPSTASH_REDIS_REST_TOKEN=your-redis-token

# أو Vercel KV
KV_REST_API_URL=https://your-kv-url.vercel-storage.com
KV_REST_API_TOKEN=your-kv-token

# كلمة مرور الداشبورد
DASHBOARD_PASSWORD=your-secure-password
```

### حدود الأمان

| الإعداد | القيمة الافتراضية | الوصف |
|---------|------------------|-------|
| `maxRequests` | 5 | الحد الأقصى للرسائل في الدقيقة |
| `maxMessagesPerDay` | 50 | الحد الأقصى للرسائل يومياً |
| `blockDuration` | 24 ساعة | مدة حظر العنوان المخالف |
| `maxMessageLength` | 10000 | الحد الأقصى لطول الرسالة |
| `maxMessageAge` | 90 يوم | عمر الرسالة القصوى |

## 📊 مراقبة الأمان

### سجلات النظام
- جميع محاولات الإرسال مسجلة
- العناوين المحظورة مسجلة
- محاولات البريد العشوائي مسجلة

### التنبيهات
- تنبيه عند تجاوز الحدود
- تنبيه عند اكتشاف بريد عشوائي
- تنبيه عند محاولات الحظر

## 🔧 تحسينات الأمان الإضافية

### 1. القوائم البيضاء والسوداء
```javascript
// في SECURITY_CONFIG
allowedDomains: ['gmail.com', 'outlook.com'], // نطاقات مسموحة فقط
blockedDomains: ['spam.com', 'malicious.com'], // نطاقات محظورة
```

### 2. حظر العناوين
```javascript
// في SECURITY_CONFIG
blockedIPs: ['192.168.1.100', '10.0.0.50'], // عناوين محظورة دائماً
```

### 3. التحقق من المرفقات
- الحد الأقصى: 1.5MB للصور
- الحد الأقصى: 2.5MB للصوت
- أنواع الملفات المسموحة: محددة

## 🚨 الاستجابة للتهديدات

### محاولة هجوم
1. **حظر فوري**: العنوان يتم حظره تلقائياً
2. **تسجيل**: جميع التفاصيل مسجلة
3. **تنبيه**: إشعار للمسؤول

### تجاوز الحدود
1. **منع مؤقت**: منع الإرسال لمدة محددة
2. **رسالة خطأ**: إبلاغ المستخدم بالسبب
3. **فرصة ثانية**: السماح بالمحاولة بعد فترة

## 📝 أفضل الممارسات

### للمطورين
1. **استخدم HTTPS دائماً**
2. **غيّر مفاتيح التشفير بانتظام**
3. **راجع السجلات يومياً**
4. **حدّث النظام بانتظام**

### للمستخدمين
1. **استخدم كلمة مرور قوية للداشبورد**
2. **لا تشارك بيانات الدخول**
3. **راجع الرسائل بانتظام**
4. **أرشف الرسائل القديمة**
